# ClawNex Risk Register

Document ID: CLAWNEX-REG-002
Version: 1.8
Date Created: 2026-04-22
Last Updated: 2026-05-15
Owner: Project owner
Purpose: Track identified product, security, operational, supply-chain, and governance risks in a structured way so that mitigations, owners, and target dates are visible and auditable.

## How this register is used

1. Every open audit finding, deferred remediation, and governance gap should map to a risk here.
2. Mitigation PRs / runbooks should reference the risk ID.
3. The register is reviewed at minimum every release and any time a new material risk is identified.

## Fields

For each risk, capture:
- Risk ID
- Title
- Category
- Description
- Likelihood (Low / Medium / High)
- Impact (Low / Medium / High / Critical)
- Priority (P0 blocks release / P1 before next release / P2 within quarter / P3 tracked)
- Owner
- Treatment (Mitigate / Accept / Avoid / Transfer)
- Target date
- Status (Open / In Progress / Mitigated / Closed / Accepted)
- Linked evidence / notes

## Categories used

- Product / Security — where product behavior itself creates risk
- Supply Chain — third-party dependencies, installers, distribution
- Dependency — version-specific exposure
- Security / LLM — model-layer attack and content risks
- Security / Compliance — where control gap affects audit posture
- Security / Operational — where control exists but depends on discipline
- Deployment — operator-facing deployment truth
- Compliance — process/documentation posture for enterprise/SOC 2/ISO
- Governance — how the project itself is run (maintainer, legal, release)
- Operational — day-to-day operation of the project or reference deployments
